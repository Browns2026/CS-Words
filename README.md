#### 1. HashCode
- 21 April, 2026
    - HashCode란 객체를 빠르게 찾기 위한 정수값(Hash Value)이다.

    예를 들어, "Apple".GetHashCode()
    를 실행하면 문자열 "Apple"에 대한 정수값이 반환된다.
    
    
    컴퓨터에서 Apple 이라는 이름의 데이터를 하나씩 비교하며 검색하는 Linear Search(순차 탐색) 다소 속도가 느릴 수 있다는 단점이 있다. 따라서 HashCode 를 사용하면 특정 데이터를 정수값으로 변환해, 저장위치를 빠르게 찾을 수 있다.

    - 사용처

    Dictionary, HashSet, Cache(로그인 캐시), DB Indexing, Double Check(중복검사) 등에서 사용된다.

    - 중요한 특징
    "Apple" 이라는 두개의 값이 있을 때, 동일한 HashCode 를 반환한다. 하지만 서로 다른 값이 동일한 HashCode 를 가질 수도 있는데, 우리는 이를 Hash Collision (해시 충돌) 이라고 한다.

    - 결론

    HashCode 는 빠른 검색/분류를 위한 기술이지 절대적인 고유 ID 는 아니다.

        - 참고 : 자료구조, Dictionary 내부 동작, DB Indexing, Cache System 에서 더 자세히 다룬다.




#### 2.CPU와 GPU의 차이
- 22 April, 2026
    - CPU(Central Processing Unit)는 컴퓨터 전반적인 연산과 제어를 담당하는 장치다
    
    기본적으로 복잡한 계산이나 판단 작업에 강하며, 운영체게 실행, 프로그램 처리, 게임 로직, 조건문 처리 등을 담당한다.

    예를 들어, 게임의 AI, 충돌판정, 프로그램 실행, 인터넷 브라우저 동작 등은 대부분 CPU 가 처리한다.

    - GPU(Graphics Processing Unit)는 대량의 단순계산을 동시에 처리하는 장치이다.

    원래는 그래픽 처리를 위해 만들어졌지만, 현재는 영상처리나 AI 연산 등 다양한 분야에서도 사용되고 있다.
    
    예를들어, 게임 그래픽 출력, 그림자 처리, 조명효과, 렌더링, AI 딥러닝 연산 등은 GPU 에서 담당한다.


    - CPU와 GPU의 차이

    | CPU | GPU |
    | --- | --- |
    | 복잡한 계산 | 단순 반복 계산 |
    | 적은 수의 고성능 코어 사용 | 많은 수의 코어 사용 |
    | 판단 및 제어 중심 | 병렬 처리 중심 |
    | 운영체제와 프로그램 관리 | 그래픽 및 대량 연산 처리 |
    

    특히 게임 개발에서는 Unreal Engine 이나 Unity 와 같은 게임 엔진에서는 CPU 와 GPU 가 함께 동작한다.
    주로 Game Engine 에서 CPU 가 담당하는 작업으로는 게임로직, 플레이어 입력 처리, 몬스터 AI, 충돌 판정, 물리 계산 등이 있으며, 
    GPU가 담당하는 작업으로는 렌더링, 배경출력, 이펙트 처리, 조명 및 그림자, 애니메니션 화면 출력 등이 있다.

    - 결론

    CPU 는 복잡한 계산과 제어를 담당하는 장치이며, GPU 는 대량의, 그래픽 및 병렬 연산을 처리하는 장치이다.
    두 장치는 서로 역할이 다르며, 현대의 프로그램과 게임에서는 CPU와 GPU 가 함께 동작하여 성능을 향상시킨다.



#### 3. 소프트웨어의 정의
- 25 April, 2026
    - 소프트웨어(Software)란, 컴퓨터가 하드웨어를 동작시키기 위한 프러그렘과 데이터의 집합이다.

    즉, 사용자가 원하는 작업을 수행할 수 있도록 컴퓨터에게 명령을 전달하는 역할을 한다.
    따라서, 소프트웨어는 눈에 보이지 않는 논리적 구성 요소이며, 하드웨어 없이 단독으로 동작할 수 없다.

    - 하드웨어와 소프트웨어의 차이
    
    | Hardware | Software |
    | --- | --- |
    | 물리적으로 존재 | 비물리적 존재 |
    | 실제 만질 수 있음 | 코드와 데이터 형태로 존재 |
    | CPU, RAM, 키보드 등 | Windows, Unity, Game 등 |
    | 시간이 지나면 마모됨 | 수정과 업데이트 가능 |

    - 소프트웨어의 종류 2가지

    | 시스템 | 응용 |
    | --- | --- |
    | 컴퓨터 자체를 동작이시키 위한 소프트웨어 | 사용자가 특정 작업을 수행하기 위한 소프트웨어 |
    | Windows, MacOs, Linux | Unity, Chrome, Photoshop, Game Program |
    
    - 소프트웨어의 역할
    하드웨어 제어, 사용자 명령 처리, 데이터 저장/관리, 프로그램 실행, 사용자와 컴퓨터 간의 상호작용 제공 등이 있다.

    - 결론
    소프트웨어는 컴퓨터 하드웨어를 동작시키기 위한 프로그램과 데이터의 집합이며, 사용자가 원하는 기능을 수행할 수 있도록 컴퓨터를 제어하는 역할을 한다.
